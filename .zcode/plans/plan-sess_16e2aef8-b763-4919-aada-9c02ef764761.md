## สาเหตุที่ Vercel deploy ไม่ผ่าน

รัน `npm run build` ในเครื่องแล้ว fail ตรงกับที่ Vercel เจอ **ไม่ใช่ปัญหา env var** — env มีครบใน `.env.local` และไม่ได้ commit ขึ้น GitHub (ดีอยู่แล้ว) สาเหตุจริงคือ **TypeScript type error** ที่ทำให้ `next build` exit code 1

### 🔴 Error หลัก (ทำให้ build fail)

`src/lib/data/index.ts:484`
```
Type error: Argument of type 'Project' is not assignable to parameter of type 'never'.
The intersection 'Task & ProgressReport & Project & ActivityLog' was reduced to 'never'
because property 'status' has conflicting types in some constituents.
```

**ต้นเหตุ:** บรรทัด 480–487 ใช้ `Promise.allSettled` โหลด 4 query แล้วกระจายผลลัพธ์ลง array แยกตาม `key` แต่การ cast `result.value as typeof projects | typeof tasks | typeof reports | typeof logs` แล้ว cast กลับเป็น `typeof value` ตอน `.push()` ทำให้ TS รวม intersection ของทั้ง 4 type ซึ่งลดเป็น `never` (เพราะ field `status` มี type ต่างกันในแต่ละ entity) → push ลง `never[]` ไม่ได้ → build fail

### 🟡 Warning (ไม่ fail แต่ควรแก้)

`src/middleware.ts` — Next.js 16 เปลี่ยนชื่อ convention จาก `middleware` เป็น `proxy` (อ้างอิง docs `node_modules/next/dist/docs/01-app/01-getting-started/16-proxy.md`)

## แผนการแก้

### 1) แก้ type error ใน `src/lib/data/index.ts` (บรรทัด 480–497)

เขียนใหม่แบบ type-safe โดยใช้ type narrowing ตามค่า `key` แทน union cast — เปลี่ยน loop `forEach` แบบเดิมที่ใช้ index เข้าถึง `entries[i]` มาเป็น `entries.map()` ที่ส่ง `key` คู่กับผลลัพธ์ แล้วจัดสาขาตาม `key`:

```ts
const results = await Promise.allSettled(entries.map((e) => e.fn()));

for (let i = 0; i < entries.length; i++) {
  const { key } = entries[i];
  const result = results[i];
  if (result.status === "fulfilled") {
    if (key === "projects") projects.push(...(result.value as Project[]));
    else if (key === "tasks") tasks.push(...(result.value as Task[]));
    else if (key === "reports") reports.push(...(result.value as ProgressReport[]));
    else logs.push(...(result.value as ActivityLog[]));
  } else {
    const raw = result.reason;
    const message =
      (raw && typeof raw === "object" && "message" in raw && String((raw as { message: unknown }).message)) ||
      (typeof raw === "string" && raw) ||
      "Unknown error";
    errors.push({ key, message });
  }
}
```

ภายในแต่ละ branch `if (key === "...")` TS จะ narrow `key` เป็น literal type ทำให้ cast `result.value as T[]` ปลอดภัยและไม่มี union ที่จะถูกลดเป็น `never` — logic การทำงานเหมือนเดิมทุกประการ

### 2) เปลี่ยน `middleware.ts` → `proxy.ts` (กำจัด warning deprecation)

ตาม Next.js 16 docs: rename ไฟล์ `src/middleware.ts` เป็น `src/proxy.ts` และเปลี่ยนชื่อ function `middleware` → `proxy` (export `config`/matcher เหมือนเดิม) เนื้อหา logic ใน `src/lib/supabase/middleware.ts` เรียกใช้จาก proxy ได้เหมือนเดิมไม่ต้องแก้

### 3) ยืนยัน

รัน `npm run build` จนผ่าน (exit 0) เพื่อยืนยันว่า Vercel จะ deploy ได้

## สิ่งที่จะเกิดขึ้นหลังแก้

- Build ผ่าน → Vercel redeploy ผ่าน
- เนื้อหา/พฤติกรรมของแอปเหมือนเดิมทุกอย่าง ไม่มีการเปลี่ยนแปลงฟังก์ชัน
- ไม่แตะ env var / config Supabase / ข้อมูล

## หมายเหตุสำหรับคุณ

หลังผมแก้เสร็จ คุณต้อง **commit + push ขึ้น GitHub** เอง Vercel ถึงจะ trigger redeploy (ผมจะไม่ push ให้ นอกจากคุณสั่ง) และต้องแน่ใจว่า env var `NEXT_PUBLIC_SUPABASE_URL` กับ `NEXT_PUBLIC_SUPABASE_ANON_KEY` ถูกตั้งไว้ใน **Vercel Project Settings → Environment Variables** ด้วย (ค่าใน `.env.local` ไม่ไปถึง Vercel)
